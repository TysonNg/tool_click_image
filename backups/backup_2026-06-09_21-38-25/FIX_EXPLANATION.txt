  QUESTION 2: "Load file but window not set"
================================================================================

YOUR PROBLEM:
  "I set target window, edit actions, save file"
  "Load file again but window not recognized"
  "Have to set window again manually"

WHAT WAS HAPPENING:
  1. You set window → save file
  2. File saved: {
       "templates": [...],
       "game_window_title": "MapleStoryM"  ← Saved!
     }
  3. Later: You load file
  4. File loaded but WINDOW NOT RESTORED
  5. You have to click "🎯 Xác Định Cửa Sổ Đích" again
  6. Extra step, waste of time

THE FIX (Auto-Restore):
  1. UPDATED: load_scenario() function
  2. UPDATED: load_scenario_combo() function
  3. NEW: When loading, check for "game_window_title"
  4. NEW: Try to find that window automatically
  5. NEW: Set it as target window
  6. NEW: Update UI to show restored window
  
RESULT:
  ✅ Window automatically restored on load
  ✅ No manual selection needed
  ✅ Clear messages if window not found
  ✅ Faster workflow

HOW IT'S IMPLEMENTED:
  Location: scenario/io.py
  
  def load_scenario():
    # Load file
    scenario = json.load(file)
    
    # Check for saved window
    saved_window_title = scenario.get("game_window_title")
    if saved_window_title:
      # Try to find window
      hwnd = find_window_by_title(saved_window_title)
      if hwnd:
        # Set as target
        state.game_hwnd = hwnd
        state.game_window_title = saved_window_title
        # Update UI
        _update_root_title()
        _update_target_window_display()
      else:
        # Show warning if not found
        show_warning("Window not found")
    
    # Update status
    state.UI.status_label.config(text="✅ Window: MapleStoryM")

WHAT YOU DO:
  OLD: Load → Select window → Run (3 steps)
  NEW: Load → Run (2 steps!)
  
  ✅ Faster!
  ✅ Fewer mistakes!
